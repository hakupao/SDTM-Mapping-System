内容：総合テスト結果（PR-0006）に基づく PR 手術関連データの Mapping 条件見直し

[背景／根拠]
- 2026年2月25日 第1版 ENSEMBLE SDTM Data Set のテストで作成されたファイル：
  「ENSEMBLE_VAPORCONEデータ移行総合テストシナリオ 202060224.xlsx」
- PR.csv テスト結果「PR-0006」にて、PRSTDTC が空白のレコードを確認（54/1752）
- PRSTDTC 空白レコードは全て手術関連データに起因（手術実施の有無／TME実施の有無／プロトコル治療中止後のTME実施の有無）
- 調査結果：
  - 「手術実施の有無」は未実施も記録対象（出力維持）
  - 「TME実施の有無」および「プロトコル治療中止後のTME実施の有無」は、なし／不明は Mapping 対象外（記録不要）

[変更内容]
- 対象ファイル：ENSEMBLE_OperationConf.xlsx（Mapping シート）
- 対象データ：TME実施／プロトコル治療中止後のTME実施
  - PROCCUR のデータ処理を「FIX」から「SEL」に変更
  - 出力条件を以下に設定
    - TME実施：NOM_TME_YNCD = "Y" の場合のみ出力
    - プロトコル治療中止後のTME実施：NNOM_TME_YNCD = "Y" の場合のみ出力

[影響]
- 04_SDTM ステップの PR.csv において、レコード数が 1752 → 1701（-51）に変更（減少分は全て TME 手術データ）
